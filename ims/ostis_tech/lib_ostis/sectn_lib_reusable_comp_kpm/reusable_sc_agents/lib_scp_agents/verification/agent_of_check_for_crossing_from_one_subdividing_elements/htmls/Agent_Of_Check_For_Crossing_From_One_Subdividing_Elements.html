Данный агент инициируется при условии появления в памяти вопросной конструкции, соответствующей действию. проверить наличие непустых пересечений у элементов одного разбиения в рамках указанной структуры. Аргументом запроса является структура, которая содержит элементы, предназначенные для проверки. В данном агенте проверяется наличие принадлежности какого-либо экземпляра к абсолютным понятиям, которые являются элементами одного и того же разбиения. Проверка на наличие такого рода ошибок происходит по всей базе знаний, но только на sc-узлах, принадлежащих входной структуре. Распишем поэтапно процесс проверки заданной структуры на наличие данного рода ошибок:

  • Выделение в заданной структуре всех sc-узлов (отсекаем sc-дуги).
  • Поиск у каждого sc-узла всех надклассов (за итерацию проверяется один sc-узел из входной структуры).
  • Поиск ситуаций, когда хотя бы два надкласса являются элементами одного разбиения. Если такие ситуации есть, то вся конструкция разбиения заносится в ответ.
Возможные результаты работы агента:
  • Если на вход агенту в качестве первого параметра пришла не структура, то генерируется сообщение об ошибке (Wrong 1st parameter! It must be a structure.).
  • Если контекст действия, поданный на вход агенту в качестве второго аргумента, не является структурой, то генерируется сообщение об ошибке (Wrong 2nd parameter! It must be a structure.).
  • Если в результате работы агента была найдены ошибки, расписанная выше, то они заносятся в структуру, содержащую элементы, подклассы которых являются элементами одного разбиения. Пример найденной ошибки представлен на рисунке.
  • Если в результате работы агента ошибки не найдены, то генерируется сообщение об отсутствии данного рода ошибок на заданной стуктуре в рамках базы знаний (Problematic knowledge base fragments are not found.).
